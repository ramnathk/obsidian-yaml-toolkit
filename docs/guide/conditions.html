Conditions ​

Conditions determine which notes will be affected by your rules.

Basic Syntax ​

yaml
IF <field> <operator> <value>

Comparison Operators ​

Equality ​

yaml
IF status = "draft"          # Equals
IF status != "archived"      # Not equals

Numeric Comparisons ​

yaml
IF count > 10                # Greater than
IF count < 5                 # Less than
IF priority >= 3             # Greater than or equal
IF priority <= 7             # Less than or equal

Date Comparisons ​

yaml
IF created_date > "2025-01-01"
IF deadline < TODAY()

Logical Operators ​

AND ​

Both conditions must be true:

yaml
IF tags contains "project" AND status = "active"

OR ​

Either condition can be true:

yaml
IF priority = "high" OR deadline < TODAY()

NOT ​

Negates a condition:

yaml
IF NOT HAS completed_date
IF NOT tags contains "archived"

Special Operators ​

contains ​

Check if an array contains a value:

yaml
IF tags contains "important"
IF categories contains "work"

HAS ​

Check if a field exists:

yaml
IF HAS custom_field
IF NOT HAS completed_date

IN ​

Check if a value is in a list:

yaml
IF status IN ["draft", "review", "published"]
IF priority IN [1, 2, 3]

Complex Conditions ​

Combine multiple operators:

yaml
IF (status = "active" OR status = "pending") AND priority > 5
IF HAS tags AND NOT tags contains "archived" AND created_date > "2025-01-01"

Field Types ​

Strings ​

yaml
IF title = "My Note"
IF status != ""

Numbers ​

yaml
IF count = 42
IF priority > 0

Booleans ​

yaml
IF published = true
IF archived != false

Arrays ​

yaml
IF tags contains "project"
IF categories = ["work", "urgent"]
