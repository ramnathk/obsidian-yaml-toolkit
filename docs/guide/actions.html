Actions ​

Actions specify what changes to make to your notes' YAML front matter.

Basic Operations ​

Set Field ​

Set or update a field value (creates or overwrites):

yaml
status = "active"
priority = 1
tags = ["project", "active"]
metadata = {author: "John", version: 2}

Add Field ​

Add a field only if it doesn't exist (safe add):

yaml
ADD created_date "2025-01-01"
ADD status "draft"

If the field already exists, no change is made.

Delete Field ​

Remove a field entirely:

yaml
DELETE old_field
DELETE temp_status

You can also use CLEAR as an alias for DELETE.

Rename Field ​

Rename a field:

yaml
RENAME old_name new_name
RENAME draft_date created_date

Array Operations ​

Add to Array ​

Add item to array (skips if already present):

yaml
ADD tags "reviewed"
ADD categories "work"

Remove from Array ​

Remove first occurrence of item:

yaml
REMOVE tags "draft"
REMOVE categories "archived"

Remove All Occurrences ​

Remove all matching items from array:

yaml
REMOVE_ALL tags "old"
REMOVE_ALL history "deprecated"

Append to Array ​

Add to end of array (even if duplicate):

yaml
APPEND history "reviewed-2025"
APPEND log TODAY()

Prepend to Array ​

Add to beginning of array:

yaml
PREPEND tags "priority"
PREPEND breadcrumbs "Home"

Insert at Position ​

Insert at specific index (0-based):

yaml
INSERT_AT tags "urgent" AT 0
INSERT_AT history "checkpoint" AT 5

Negative indices count from end: AT -1 inserts before last item.

Insert After Value ​

Insert after first occurrence of target value:

yaml
INSERT_AFTER tags "new-tag" AFTER "existing-tag"
INSERT_AFTER categories "sub-cat" AFTER "main-cat"

Insert Before Value ​

Insert before first occurrence of target value:

yaml
INSERT_BEFORE tags "prefix" BEFORE "main-tag"
INSERT_BEFORE steps "prerequisite" BEFORE "first-step"

Remove at Position ​

Remove item at specific index:

yaml
REMOVE_AT tags 0
REMOVE_AT history -1

Negative indices count from end: -1 removes last item.

Replace Value ​

Replace first occurrence:

yaml
REPLACE tags "old-value" WITH "new-value"
REPLACE status "draft" WITH "review"

Replace All Values ​

Replace all occurrences:

yaml
REPLACE_ALL tags "old" WITH "new"
REPLACE_ALL categories "deprecated" WITH "current"

Deduplicate Array ​

Remove duplicate values:

yaml
DEDUPLICATE tags
DEDUPLICATE categories

Sort Array ​

Sort array in ascending or descending order:

yaml
SORT tags ASC
SORT priorities DESC
SORT categories

Default order is ASC if not specified.

Sort Objects by Field ​

Sort array of objects by a field:

yaml
SORT_BY tasks BY priority DESC
SORT_BY entries BY date ASC
SORT_BY items BY name

Move Item ​

Move item from one index to another:

yaml
MOVE tags FROM 0 TO 5
MOVE history FROM -1 TO 0

Move Items by Condition ​

Move items matching condition to start, end, or relative position:

yaml
MOVE_WHERE tags WHERE value = "urgent" TO START
MOVE_WHERE tasks WHERE priority > 5 TO END
MOVE_WHERE items WHERE status = "done" AFTER status = "active"
MOVE_WHERE entries WHERE type = "note" BEFORE type = "summary"

Update Items by Condition ​

Update fields in objects matching condition:

yaml
UPDATE_WHERE tasks WHERE status = "pending" SET status "active", started TODAY()
UPDATE_WHERE items WHERE priority < 3 SET priority 5
UPDATE_WHERE entries WHERE type = "draft" SET type "review", reviewed true

Object Operations ​

Merge Objects ​

Deep merge objects (preserves existing nested fields):

yaml
MERGE metadata {author: "John", version: 2}
MERGE config {theme: "dark", showIcons: true}

Nested objects are recursively merged. Arrays and primitives are replaced.

Merge Overwrite ​

Shallow merge objects (overwrites all fields):

yaml
MERGE_OVERWRITE metadata {author: "Jane", version: 3}
MERGE_OVERWRITE settings {mode: "advanced"}

All fields from the new object replace existing fields at the top level only.

Special Values ​

Functions ​

Available in any action:

  • NOW() - Current timestamp (ISO 8601 format)
  • TODAY() - Today's date (YYYY-MM-DD)
  • UUID() - Generate unique identifier
yaml
created_date = NOW()
id = UUID()
updated = TODAY()

Data Types ​

Supported value types:

  • Strings: "text" (use quotes)
  • Numbers: 42, 3.14
  • Booleans: true, false
  • Null: null
  • Arrays: ["item1", "item2"]
  • Objects: {key: "value", count: 5}
  • Dates: "2025-01-01"

Multiple Actions ​

Execute multiple actions in one rule (comma-separated):

yaml
status = "completed", completed_date = NOW(), ADD tags "done"

Actions execute left-to-right in order.
