0. YAML Syntax Guide ‚Äã

When Strings Need Quotes ‚Äã

‚úÖ NO QUOTES needed for simple strings:

yaml
title: My Note
status: draft
name: Website Redesign
date: 2025-11-19

‚ö†Ô∏è QUOTES REQUIRED when value contains:

yaml
# Colons followed by space
note: "Key: Value"

# Square brackets (arrays in string)
link: "[[Wikilink]]"

# Curly braces (objects in string)
template: "{{variable}}"

# Quotes inside value (use escaping)
quote: "He said \"hello\""
# OR use single quotes
quote: 'He said "hello"'

# Leading special characters
mention: "@username"
command: "#hashtag in value"

# Values that look like other types
version: "1.0"      # Without quotes = number 1.0
flag: "true"        # Without quotes = boolean true
empty: "null"       # Without quotes = null value

# Leading/trailing whitespace to preserve
padded: "  spaced  "

# Multiline strings with preserved newlines
description: |
  Line 1
  Line 2
  Line 3

Arrays and Objects:

yaml
# Simple arrays - bracket notation (same as test-vault)
tags: [work, urgent, project]
aliases: [Overview, Project Summary]

# Array of objects - list notation (same as test-vault)
countsLog:
  - book: "The Hobbit"
    count: 108
  - book: "1984"
    count: 54

# Nested objects - indented notation (same as test-vault)
metadata:
  author: John Doe
  version: 1.0
  created: 2025-11-01

# Alternative: Arrays can also use list notation
tags:
  - work
  - urgent
  - project

# Alternative: Objects can use inline notation (compact)
metadata: {author: John, version: 1.0}

Special Characters in Unquoted Strings:

yaml
# ‚úÖ These are OK without quotes
book: ‡•ê ‡§®‡§Æ‡§É ‡§∂‡§ø‡§µ‡§æ‡§Ø    # Unicode OK
file-name: my-note      # Hyphens OK
number_id: 12345        # Looks like number but treated as string

# ‚ùå These NEED quotes
mood: "üòä"              # Emoji should be quoted
path: "C:\\Users\\John" # Backslashes need escaping

YAML Spec Compliance Notes ‚Äã

  • Indentation: 2 spaces (no tabs)
  • List items: Start with - (hyphen + space)
  • Key-value: key: value (colon + space)
  • Comments: Start with #
  • Booleans: true, false, yes, no, on, off
  • Null: null, ~, or empty value
  • Numbers: Integers and floats auto-detected
  • Strings: Auto-quoted in output when necessary

2. Conditional Operations ‚Äã

2.1 Basic Comparisons ‚Äã

Example 2.1.1: Equality

yaml
# Input files:
# note1.md: status: draft
# note2.md: status: published
# note3.md: (no status field)

# Rule
Condition: status = "draft"
Action: SET reviewed true

# Output
# note1.md: status: draft, reviewed: true (MATCHED)
# note2.md: status: published (NO CHANGE)
# note3.md: (NO CHANGE)

Example 2.1.2: Inequality

yaml
# Rule
Condition: status != "archived"
Action: SET active true

# Matches all files where status is NOT "archived"
# Also matches files without status field

yaml
# Input
---
priority: 5
---

# Rule

Action: SET urgent true

# Output
---
priority: 5
urgent: true
---

4. Object Operations ‚Äã

4.1 MERGE - Merge objects (deep) ‚Äã

Example 4.1.1: MERGE simple object

yaml
# Input
---
metadata:
  author: John
  version: 1.0
---

# Rule
Action: FOR metadata MERGE { "editor": "Jane", "reviewed": true }

# Output
---
metadata:
  author: John
  version: 1.0
  editor: Jane
  reviewed: true
---

Example 4.1.2: MERGE overwrites existing keys

yaml
# Input
---
metadata:
  author: John
  version: 1.0
---

# Rule
Action: FOR metadata MERGE { "version": "2.0", "status": "published" }

# Output
---
metadata:
  author: John
  version: 2.0
  status: published
---

Example 4.1.3: MERGE deep (nested objects)

yaml
# Input
---
config:
  ui:
    theme: dark
    fontSize: 14
  data:
    cache: true
---

# Rule
Action: FOR config MERGE { "ui": { "fontSize": 16, "fontFamily": "Arial" } }

# Output (ui.fontSize updated, ui.theme preserved, ui.fontFamily added)
---
config:
  ui:
    theme: dark
    fontSize: 16
    fontFamily: Arial
  data:
    cache: true
---

Example 4.1.4: MERGE with arrays (arrays REPLACED, not merged)

yaml
# Input
---
data:
  tags: [a, b, c]
  count: 5
---

# Rule
Action: FOR data MERGE { "tags": ["x", "y"], "count": 10 }

# Output (tags array REPLACED)
---
data:
  tags: ["x", "y"]
  count: 10
---

4.2 MERGE_OVERWRITE - Shallow merge (replaces nested objects) ‚Äã

Example 4.2.1: MERGE_OVERWRITE vs MERGE

yaml
# Input
---
config:
  ui:
    theme: dark
    fontSize: 14
---

# Rule (MERGE)
Action: FOR config MERGE { "ui": { "fontSize": 16 } }

# Output (theme preserved - deep merge)
---
config:
  ui:
    theme: dark
    fontSize: 16
---

# Rule (MERGE_OVERWRITE)
Action: FOR config MERGE_OVERWRITE { "ui": { "fontSize": 16 } }

# Output (theme lost - shallow merge)
---
config:
  ui:
    fontSize: 16
---

5.4 Boolean Logic - Parentheses ‚Äã

Example 5.4.1: Precedence with parentheses

yaml
# Rule (without parentheses - AND has higher precedence)
Condition: status = "draft" OR priority > 5 AND tags has "urgent"

# Parsed as: status = "draft" OR (priority > 5 AND tags has "urgent")

# Rule (with parentheses - explicit grouping)
Condition: (status = "draft" OR priority > 5) AND tags has "urgent"

# Parsed as: (status = "draft" OR priority > 5) AND tags has "urgent"

Example 5.4.2: Complex nested conditions

yaml
# Input
---

priority: 8
tags: [work, urgent]
reviewed: false
---

# Rule

Action: SET escalate true

# Output (all conditions match)
---

priority: 8
tags: [work, urgent]
reviewed: false
escalate: true
---
