Syntax Reference ​

Complete syntax reference for YAML Manipulator rules.

Rule Structure ​

A rule has two parts entered in separate fields:

Condition field: What to match

<condition>

Action field: What to do

<action> [, <action>, ...]

Important: Don't type "IF" or "THEN" - those are handled by the plugin automatically!

Conditions ​

Comparison Operators ​

  • = - Equals
  • != - Not equals
  • > - Greater than
  • < - Less than
  • >= - Greater than or equal
  • <= - Less than or equal

Condition examples:

status = "active"
count > 10
date >= "2025-01-01"

Logical Operators ​

  • AND - Both conditions must be true
  • OR - Either condition can be true
  • NOT - Negates a condition

Examples:

tags contains "project" AND status != "archived"
priority = "high" OR deadline < NOW()
NOT HAS completed_date

Special Operators ​

  • contains - Check if array contains value
  • HAS - Check if field exists
  • IN - Check if value is in list

Examples:

tags contains "important"
HAS custom_field
status IN ["draft", "review", "published"]

Actions ​

Set Field ​

Set or update a field value:

yaml
status = "active"
priority = 1
tags = ["project", "active"]

Add to Array ​

Add item to an array (if not already present):

yaml
ADD tags "reviewed"
ADD categories "work"

Remove from Array ​

Remove item from an array:

yaml
REMOVE tags "draft"
REMOVE categories "archived"

Delete Field ​

Remove a field entirely:

yaml
DELETE old_field
DELETE temp_status

Append to Array ​

Append item even if it already exists:

yaml
APPEND history "reviewed-2025"
APPEND log TODAY()

Multiple Top-Level Fields ​

One rule = One action. To set multiple top-level fields, create multiple rules with the same condition:

yaml
# Rule 1
Condition: status = "pending"
Action: SET status "completed"

# Rule 2
Condition: status = "pending"
Action: SET completed_date "2025-11-24T22:10:00"

Array operations support comma-separated field updates with UPDATE_WHERE:

yaml
UPDATE_WHERE tasks WHERE status = "pending" SET status "active", priority 1

Special Values ​

Functions ​

  • NOW() - Current timestamp (ISO 8601)
  • TODAY() - Today's date (YYYY-MM-DD)
  • UUID() - Generate unique ID

Examples:

yaml
created_date = NOW()
id = UUID()
updated = TODAY()

Data Types ​

  • Strings: Use quotes - "text"
  • Numbers: No quotes - 42, 3.14
  • Booleans: true, false
  • Arrays: ["item1", "item2"]
  • Objects: {key: "value", nested: {foo: "bar"}}
  • Dates: ISO format - "2025-01-01"

Complete Examples ​

Update draft notes ​

Condition:

status = "draft" AND HAS tags

Action:

status = "ready", reviewed_date = NOW()

Clean up archived notes ​

Condition:

tags contains "archived"

Action:

DELETE priority, DELETE deadline, status = "archived"

Add missing metadata ​

Condition:

NOT HAS created_date

Action:

created_date = NOW(), status = "new"

Conditional tagging ​

Condition:

priority > 5 AND NOT tags contains "urgent"

Action:

ADD tags "urgent"
